# AGENTS.md

이 저장소는 코드잇 부트캠프 기간 동안의 학습 기록을 정리하는 저장소입니다. 앞으로 이 저장소에서 TIL, WIL, learning 문서를 작성하거나 수정할 때는 아래 규칙을 따릅니다.

## 저장소 목적

- 코드잇 부트캠프 기간 동안의 학습 기록을 정리합니다.
- TIL, WIL, 예습/복습, 과제, 알고리즘, 프로젝트 회고를 기록합니다.
- 강의 내용을 그대로 복사하지 않고, 내가 이해한 내용과 어려웠던 점, 해결 방법을 중심으로 작성합니다.
- 취업 준비용 포트폴리오로도 활용할 수 있지만, 우선 목적은 나의 성장 과정을 기록하는 것입니다.

## 폴더 구조

- `daily/`: 매일 작성하는 TIL
- `weekly/`: 매주 작성하는 WIL
- `learning/`: 주제별 상세 학습 정리
- `assignments/`: 과제 정리
- `algorithm/`: 알고리즘 문제 풀이 기록
- `projects/`: 프로젝트 회고 및 진행 기록

## TIL 작성 규칙

TIL 파일 위치는 다음 형식을 사용합니다.

```text
daily/YYYY-MM/YYYY-MM-DD.md
```

예시:

```text
daily/2026-05/2026-05-05.md
```

TIL은 아래 구조로 작성합니다.

```markdown
# [TIL] YYYY-MM-DD

## 오늘 한 일

- 

## 오늘 배운 것

### 주제명

- 

## 어려웠던 점

- 

## 해결 방법

- 

## 내일 할 일

- 
```

TIL은 하루 전체 학습 내용을 요약하는 문서입니다. 너무 길게 작성하지 않고 핵심만 정리합니다.

## learning 작성 규칙

learning 파일 위치는 다음 형식을 사용합니다.

```text
learning/카테고리/YYYY-MM-DD-주제명.md
```

예시:

```text
learning/java/2026-05-05-java-thread-and-collection.md
learning/git/2026-05-01-git-github-and-commit-management.md
```

learning은 TIL보다 자세하게 작성합니다. 다만 강의 내용을 그대로 옮기지 않고 아래 기준으로 정리합니다.

- 핵심 개념
- 내가 이해한 내용
- 헷갈린 부분
- 예제 코드 흐름
- 주의할 점
- 다음에 다시 볼 내용

## WIL 작성 규칙

WIL 파일 위치는 다음 형식을 사용합니다.

```text
weekly/YYYY-MM/weekNN.md
```

예시:

```text
weekly/2026-05/week01.md
```

WIL은 아래 구조로 작성합니다.

```markdown
# [WIL] N주차

## 기간

- YYYY-MM-DD ~ YYYY-MM-DD

## 이번 주에 한 일

- 

## What I Learned

- 

## START

- 

## STOP

- 

## CONTINUE

- 

## 다음 주 계획

- 
```

## 문체 규칙

- 한국어로 작성합니다.
- 너무 딱딱하지 않게 작성하되, 기록용 문서처럼 정리합니다.
- 문장은 "~했다", "~라고 이해했다", "~가 필요하다고 느꼈다" 형태를 사용합니다.
- 강의 내용을 그대로 복사하지 않습니다.
- 이해한 내용을 내 말로 다시 정리합니다.
- 너무 긴 코드블록은 피합니다.
- 마크다운 코드블록이 깨지지 않도록 주의합니다.
- 코드블록 안에 다시 코드블록을 중첩하지 않습니다.
- 기존 파일을 수정할 때는 마크다운 구조가 깨지지 않도록 확인합니다.

## TIL과 learning 분리 기준

- TIL: 오늘 학습한 전체 흐름 요약
- learning: 특정 주제에 대한 상세 정리

예시:

- TIL: Java Part 3 - 상을 학습했다.
- learning: 스레드, 동기화, 제네릭, 컬렉션을 상세 정리한다.

## 작업 방식

- 사용자가 학습 메모를 주면 먼저 TIL을 작성합니다.
- 사용자가 learning도 요청하면 별도 learning 파일을 작성합니다.
- 기존 파일이 있으면 덮어쓰기 전에 어떤 파일을 수정할지 먼저 알려줍니다.
- 마크다운 문법 오류가 있으면 수정합니다.
- 커밋 메시지는 `docs:` prefix를 사용합니다.
- 수정 후에는 어떤 파일을 생성하거나 변경했는지 요약합니다.

## 커밋 메시지 예시

- `docs: 5월 5일 TIL 및 Java 스레드/컬렉션 학습 정리 추가`
- `docs: 5월 6일 TIL 및 Java 파일 I/O/소켓 학습 정리 추가`
- `docs: README 부트캠프 과정명 수정`
- `docs: Git 학습 정리 코드블록 오류 수정`
