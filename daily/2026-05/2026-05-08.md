## 오늘 배운 것

### Java static과 final

- `static` 멤버는 객체마다 따로 생기는 값이 아니라 클래스 기준으로 공유되는 값이라고 이해했다.
- `static` 메서드에서는 객체가 생성되지 않았을 수 있으므로 `this`를 사용할 수 없다.
- 모든 객체가 공유해도 되는 값이나 기능에만 `static`을 붙여야 한다고 느꼈다.
- `final`은 값이나 동작이 바뀌지 않도록 제한할 때 사용하며, 상수는 보통 `static final`로 선언한다.

### Java 상속과 다형성

- 상속은 부모 클래스의 필드와 메서드를 자식 클래스가 물려받아 코드 중복을 줄이는 방식이다.
- Java는 클래스의 단일 상속만 지원하며, 상속에는 `extends`를 사용한다.
- 오버라이딩은 부모에게 물려받은 메서드를 자식 클래스의 상황에 맞게 다시 정의하는 것이다.
- 다형성은 부모 타입의 참조 변수로 자식 객체를 다룰 수 있게 해주는 객체지향의 핵심 개념이라고 이해했다.
- 자식 고유 기능을 사용해야 할 때는 타입 캐스팅과 `instanceof` 확인이 필요하다.

### 추상 클래스와 인터페이스

- 추상 클래스는 아직 완성되지 않은 메서드를 포함할 수 있으며, 자식 클래스가 이를 구현해야 한다.
- 추상 메서드는 선언만 있고 실행 블록이 없는 메서드이다.
- 인터페이스는 여러 클래스가 같은 기능 이름과 사용 방식을 갖도록 약속을 정하는 역할을 한다.
- 추상 클래스는 공통 코드 재사용과 확장에 가깝고, 인터페이스는 구현해야 할 기능의 규칙을 강제하는 데 가깝다고 정리했다.
