## 오늘 배운 것

### UML classDiagram

- UML classDiagram은 클래스, 인터페이스, enum, 상속, 구현, 연관 관계를 시각적으로 표현하는 데 사용한다.
- `LearningActivity`는 공통 필드와 공통 메서드를 가진 추상 클래스이고, `LectureLog`, `PracticeLog`, `ReadingLog`가 이를 상속한다.
- `Reviewable`, `Shareable`은 인터페이스이며, 각 활동 클래스가 필요한 기능을 구현한다.
- `ActivityCategory`, `Visibility`는 enum으로 고정된 값의 목록을 표현한다.
- `ActivityPrinter` 인터페이스를 두고 `ConsoleActivityPrinter`, `CompactActivityPrinter`가 구현하도록 하여 출력 방식을 교체할 수 있게 구성했다.
- `ActivityDashboard`는 `Summary`, `ReportBuilder`를 내부에 포함하고, `ReportBuilder`는 `ActivityPrinter`에 의존하여 출력 책임을 분리한다.

### UML sequenceDiagram

- sequenceDiagram은 객체들이 어떤 순서로 메시지를 주고받는지 표현한다.
- `SprintLogApp`이 `ActivityDashboard`를 생성하고, `ReportBuilder`를 만든 뒤 `print()`를 호출하는 흐름을 정리했다.
- `ReportBuilder`는 `Dashboard`에 `summarize()`를 요청하고, 이후 activity 배열을 순회하며 `ActivityPrinter.print(activity)`를 호출한다.
- classDiagram이 구조를 보여준다면, sequenceDiagram은 실행 흐름을 보여준다고 이해했다.

### 제네릭

- 제네릭은 클래스나 메서드를 특정 타입에 고정하지 않고, 사용할 때 타입을 정할 수 있게 해준다.
- 제네릭을 사용하면 코드 중복을 줄이면서도 타입 안전성을 지킬 수 있다.
- 타입 매개변수는 `T`, `E`, `K`, `V` 같은 이름을 관례적으로 사용한다.
- 타입 인자로 기본형은 사용할 수 없고, `int` 대신 `Integer` 같은 래퍼 클래스를 사용해야 한다.
- 타입 매개변수에 `extends`를 사용하면 특정 타입과 그 자식 타입으로 제한할 수 있다.
- 제네릭 메서드는 메서드 호출 시점에 타입이 정해지는 메서드이다.
- 타입 이레이저는 컴파일 이후 제네릭 타입 정보가 제거되고 필요한 경우 컴파일러가 캐스팅을 추가하는 방식으로 이해했다.

### 컬렉션 프레임워크

- 컬렉션 프레임워크는 여러 데이터를 저장하고 다루기 위한 표준 자료구조와 인터페이스를 제공한다.
- `List`는 순서가 있고 중복을 허용하는 자료구조이다.
- `ArrayList`는 내부적으로 배열을 사용하며, 인덱스 조회가 빠르지만 중간 삽입/삭제는 데이터 이동이 필요하다.
- `LinkedList`는 노드가 서로 연결된 구조이며, 중간 삽입/삭제에 유리할 수 있지만 인덱스 접근은 순차 탐색이 필요하다.
- `Set`은 중복을 허용하지 않는 자료구조이다.
- `HashSet`은 해시 알고리즘을 활용해 평균적으로 빠른 검색을 제공한다.
- 해시 자료구조에서 직접 만든 객체를 사용할 때는 `equals()`와 `hashCode()`를 함께 재정의해야 한다.
- `Map`은 key-value 쌍으로 데이터를 저장하며, key는 중복될 수 없고 value는 중복될 수 있다.
- Stack은 LIFO 구조, Queue는 FIFO 구조이며, Java에서는 Stack 클래스보다 `Deque`와 `ArrayDeque` 사용을 권장한다.
- `Iterable`과 `Iterator`는 자료구조의 내부 구조를 몰라도 일관된 방식으로 순회할 수 있게 해준다.
- 정렬은 기본 정렬 기준이 필요하면 `Comparable`, 별도 정렬 기준이 필요하면 `Comparator`를 사용할 수 있다.

- 제네릭에서 타입 매개변수와 타입 인자의 차이를 구분하는 데 시간이 필요했다.

## 해결 방법

- 추상 클래스는 공통 상태와 기능을 묶는 용도, 인터페이스는 구현해야 할 역할을 정의하는 용도로 정리했다.
- 제네릭은 “사용할 타입을 나중에 정해서 코드 재사용과 타입 안전성을 함께 얻는 문법”으로 이해했다.
