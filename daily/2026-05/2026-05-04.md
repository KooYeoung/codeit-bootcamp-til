## 오늘 배운 것

### 객체지향 프로그래밍

- 객체지향 프로그래밍은 프로그램을 객체 단위로 구성하는 방식이다.
- 객체는 관련 있는 데이터와 기능을 하나로 묶은 단위이다.
- Java에서는 클래스를 사용해 객체를 정의한다.
- 클래스는 객체를 만들기 위한 설계도이자 사용자 정의 자료형으로 볼 수 있다.

### 클래스, 객체, 인스턴스

- 클래스는 객체를 표현하기 위한 문법이다.
- 객체는 실제로 다루려는 대상이다.
- 인스턴스는 클래스를 바탕으로 생성된 실제 객체를 의미한다.
- Java에서 클래스 형식의 변수는 실제 객체를 직접 담는 것이 아니라 객체에 대한 참조를 담는다.
- 참조형 변수는 `null`을 가질 수 있으며, `null` 상태에서 멤버에 접근하면 `NullPointerException`이 발생할 수 있다.

### 클래스의 구성 요소

- 클래스는 필드, 메서드, 생성자로 구성된다.
- 필드는 객체가 가지는 데이터이다.
- 메서드는 객체가 수행할 수 있는 기능이다.
- 생성자는 객체가 생성될 때 호출되어 초기화를 담당한다.
- 필드는 보통 `private`으로 숨기고, 필요한 경우 getter와 setter를 통해 접근한다.

### 생성자

- 생성자는 객체가 생성되는 시점에 자동으로 호출된다.
- 생성자의 이름은 클래스 이름과 동일하다.
- 생성자는 반환 자료형이 없다.
- 생성자를 직접 작성하지 않으면 컴파일러가 기본 생성자를 추가할 수 있다.
- 생성자는 다중 정의할 수 있으며, 매개변수 구성이 다르면 여러 생성자를 만들 수 있다.
- 생성자 내부에서 다른 생성자를 호출할 때는 `this()`를 사용할 수 있다.

### 접근 제어 지시자

- 접근 제어 지시자는 클래스나 멤버에 접근할 수 있는 범위를 제어한다.
- `public`은 외부 접근을 허용한다.
- `protected`는 같은 패키지와 자식 클래스에서 접근할 수 있다.
- `default`는 같은 패키지 안에서만 접근할 수 있다.
- `private`은 클래스 내부에서만 접근할 수 있다.
- 접근 제어는 객체를 사용하는 입장에서 불필요한 부분을 감추고, 사용해야 하는 기능만 제공하기 위해 필요하다.

### getter와 setter

- getter와 setter는 `private` 필드에 안전하게 접근하기 위한 메서드이다.
- getter는 값을 읽을 때 사용한다.
- setter는 값을 변경할 때 사용한다.
- 필드에 직접 접근하지 않고 메서드를 통해 접근하면, 값 검증이나 접근 제한을 적용할 수 있다.

### this

- `this`는 현재 객체 자신을 가리키는 참조자이다.
- 필드와 지역변수의 이름이 같을 때 `this`를 사용하면 객체의 필드임을 명확히 할 수 있다.
- 같은 클래스에서 생성된 객체라도 각 객체의 `this`는 서로 다른 인스턴스를 가리킨다.
- `static` 메서드에서는 객체 인스턴스가 없기 때문에 `this`를 사용할 수 없다.

### static

- `static` 멤버는 인스턴스가 없어도 클래스 기준으로 존재한다.
- `static` 필드는 모든 인스턴스가 공유한다.
- `static` 메서드는 인스턴스 없이 호출할 수 있다.
- `static` 메서드에서는 `this`를 사용할 수 없고, 인스턴스 필드에 직접 접근할 수 없다.
- `static final`은 상수를 정의할 때 자주 사용된다.

### 얕은 복사와 깊은 복사

- 얕은 복사는 참조 대상은 그대로 두고 참조값만 복사하는 방식이다.
- 얕은 복사를 하면 두 참조자가 같은 객체를 바라보게 되어, 한쪽 변경이 다른 쪽에도 영향을 줄 수 있다.
- 깊은 복사는 참조 대상 객체까지 새로 만들어 값을 복사하는 방식이다.
- 참조형 필드를 가진 객체를 복사할 때는 얕은 복사로 인한 사이드 이펙트를 주의해야 한다.
- 깊은 복사가 필요한 경우 복사 생성자나 별도 복사 메서드를 만들 수 있다.

### 상속

- 상속은 기존 클래스를 확장하여 새로운 클래스를 만드는 문법이다.
- 부모 클래스, 자식 클래스 또는 기본 클래스, 파생 클래스라고 부른다.
- Java에서는 `extends`를 사용해 상속한다.
- Java의 모든 클래스는 최상위에서 `Object`를 상속한다.
- Java는 클래스의 다중 상속을 허용하지 않는다.
- 상속은 `is-a` 관계로 설명할 수 있다.
- 자식 클래스 객체가 생성될 때 부모 클래스 객체도 함께 생성된다.

### super

- `super`는 부모 클래스의 멤버나 생성자를 가리킬 때 사용한다.
- 자식 클래스 생성자에서 부모 클래스 생성자를 명시적으로 호출하려면 `super()`를 사용한다.
- 부모 생성자 호출은 자식 생성자 내부에서 가장 먼저 실행되어야 한다.
- 상속 관계에서는 부모 생성자가 먼저 실행되고, 그 다음 자식 생성자의 내용이 실행된다.

### 오버라이딩

- 오버라이딩은 부모 클래스의 메서드를 자식 클래스에서 재정의하는 것이다.
- 오버로딩은 같은 이름의 메서드를 매개변수 구성을 다르게 하여 여러 개 정의하는 것이다.
- 오버라이딩과 오버로딩은 서로 다른 개념이다.
- 오버라이딩할 때는 `@Override` 어노테이션을 사용하면 실수를 줄일 수 있다.
- 오버라이딩된 메서드는 실제 인스턴스 타입을 기준으로 호출된다.
- 부모 클래스의 기능을 유지하면서 일부 로직을 추가하고 싶을 때 `super.메서드명()`을 사용할 수 있다.

### 객체 간 관계와 UML

- 클래스는 다른 클래스를 필드로 가질 수 있다.
- 객체 간 관계에는 상속, 의존, 연관, 집합, 합성 등이 있다.
- 클래스 설계는 객체와 객체 사이의 관계를 정의하는 과정이다.
- UML은 클래스와 객체 간 관계를 시각적으로 표현하기 위한 모델링 언어이다.
- 클래스 다이어그램과 시퀀스 다이어그램은 객체지향 설계에서 알아두면 좋다.

### 다형성

- 다형성은 하나의 부모 타입으로 여러 자식 타입 객체를 다룰 수 있는 객체지향의 특징이다.
- 부모 타입 참조 변수는 자식 클래스 인스턴스를 참조할 수 있다.
- 이를 업캐스팅이라고 한다.
- 부모 타입으로 참조하더라도 오버라이딩된 메서드는 실제 객체 타입 기준으로 실행된다.
- 다형성을 사용하면 코드 구조를 더 단순하고 유연하게 만들 수 있다.

### 다운캐스팅과 instanceof

- 다운캐스팅은 부모 타입 참조를 자식 타입으로 변환하는 것이다.
- 잘못된 다운캐스팅은 런타임 오류를 발생시킬 수 있다.
- `instanceof`를 사용하면 객체가 특정 타입인지 확인할 수 있다.
- 다운캐스팅이 필요한 경우에는 먼저 `instanceof`로 안전성을 확인하는 것이 좋다.
- 다만 부모 클래스가 자식 클래스 타입을 직접 판단하는 구조는 의존성이 역전될 수 있으므로 주의해야 한다.

### 추상 클래스와 인터페이스

- 추상 클래스는 직접 인스턴스를 만들 수 없는 클래스이다.
- 추상 클래스는 자식 클래스를 위한 설계 기준 역할을 한다.
- 추상 메서드는 자식 클래스에서 반드시 구현해야 한다.
- 인터페이스는 구현 없이 메서드의 형식을 정의하는 역할을 한다.
- 클래스는 인터페이스를 `implements`로 구현한다.
- 인터페이스는 다중 구현이 가능하다.
- 추상 클래스와 인터페이스는 공통 규칙을 정의하고, 구현 클래스들이 각자 세부 동작을 구현하도록 만들 때 사용한다.

### enum

- `enum`은 관련 있는 상수들을 하나의 타입으로 묶어 표현하는 문법이다.
- `enum`을 사용하면 단순 숫자나 문자열 상수보다 가독성과 타입 안정성을 높일 수 있다.
- `values()`를 사용하면 모든 enum 값을 순회할 수 있다.
- `valueOf()`를 사용하면 문자열로 enum 값을 찾을 수 있다.
- `name()`은 enum 상수 이름을 반환한다.
- `ordinal()`은 enum 상수의 순서를 반환한다.

### JVM 기본 이론

- Java 객체의 메모리 해제는 개발자가 직접 하지 않고 JVM의 Garbage Collector가 담당한다.
- JVM은 Class Loader, Runtime Data Area, Execution Engine, Garbage Collector 등으로 구성된다.
- 클래스 로딩은 Loading, Linking, Initialization 단계를 거친다.
- Runtime Data Area에는 Method Area, Heap Area, Stack Area, PC Register, Native Method Stack 등이 있다.
- 객체는 Heap 영역에 생성된다.
- 메서드 호출 시 지역변수와 매개변수 등은 Stack 영역과 관련된다.
- 정적 정보와 클래스 메타 정보는 Method Area와 관련된다.

### String

- Java에서 문자열은 `String` 클래스로 다룬다.
- `String`은 불변 객체이다.
- 문자열을 비교할 때 `==`는 참조 비교이고, `equals()`는 내용 비교이다.
- 문자열 내용 비교에는 보통 `equals()`를 사용해야 한다.
- 문자열 상수는 JVM에서 문자열 상수 풀을 통해 관리된다.
- 같은 문자열 리터럴은 같은 String 인스턴스를 참조할 수 있다.
- `intern()`은 문자열 상수 풀에서 문자열을 조회하고 반환하는 메서드이다.
- 문자열을 반복적으로 변경해야 하는 경우 `String` 덧셈은 비효율적일 수 있다.

### 예외 처리

- 예외 처리는 프로그램 실행 중 발생할 수 있는 오류 상황을 처리하기 위한 문법이다.
- `try`, `catch`, `finally`를 사용해 예외를 처리할 수 있다.
- `try` 블록에는 예외가 발생할 수 있는 코드를 작성한다.
- `catch` 블록에서는 발생한 예외를 처리한다.
- `finally` 블록은 예외 발생 여부와 관계없이 실행된다.
- `NullPointerException`은 `null` 참조로 멤버에 접근할 때 발생할 수 있다.
- 배열 범위를 벗어난 접근, 잘못된 형 변환, `null` 접근 등은 런타임 예외의 원인이 될 수 있다.

- 클래스, 객체, 인스턴스의 차이가 비슷하게 느껴져 처음에는 구분이 어려웠다.
- 상속에서 부모 생성자와 자식 생성자의 호출 순서가 헷갈렸다.
- JVM의 클래스 로딩 과정과 Runtime Data Area 구조가 추상적으로 느껴졌다.

## 해결 방법

- 클래스는 설계도, 객체는 구현 대상, 인스턴스는 클래스로부터 생성된 실제 객체로 정리했다.
- 상속 관계에서는 부모 객체가 먼저 초기화되고, 그 뒤 자식 객체가 초기화된다고 정리했다.
- JVM 클래스 로딩 과정은 `.class` 파일을 JVM이 읽어 실행 가능한 상태로 준비하는 흐름으로 정리했다.
- Runtime Data Area는 실행 중 데이터를 저장하는 메모리 구조로 보고, `Method Area`, `Heap`, `Stack`의 역할을 중심으로 이해했다.
- `new`로 생성된 객체는 Heap에 저장되고, 메서드 호출과 지역변수는 Stack과 관련된다고 정리했다.
