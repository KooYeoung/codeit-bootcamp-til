## 오늘 배운 것

### 알고리즘 기초

- 선형 탐색은 배열을 앞에서부터 순서대로 확인하는 방식이며, 정렬 여부와 상관없이 사용할 수 있다고 이해했다.
- 이진 탐색은 정렬된 데이터에서 가운데 값을 기준으로 탐색 범위를 절반씩 줄이는 방식이다.
- 선택 정렬은 아직 정렬되지 않은 구간에서 가장 작은 값의 위치를 찾고, 현재 위치와 바꾸는 방식이다.
- 시간 복잡도는 입력 크기가 커질 때 실행 시간이 얼마나 증가하는지 표현하는 기준이다.
- Big-O 표기법에서는 보통 가장 영향이 큰 항만 남겨 알고리즘의 증가 경향을 본다고 이해했다.

### Java 배열과 메서드

- 배열은 같은 타입의 여러 값을 하나의 묶음으로 관리하는 자료구조이다.
- 배열의 인덱스는 0부터 시작하며, 배열의 길이는 `length`로 확인한다.
- 2차원 배열은 배열 안에 배열이 들어 있는 구조로 이해했다.
- 메서드는 반복되는 코드 블록에 이름을 붙여 재사용하고, 코드를 더 구조적으로 만들기 위해 사용한다.
- `return`은 값을 반환할 때뿐 아니라 메서드 실행을 중간에 종료할 때도 사용할 수 있다.

### Java 객체지향 기초

- 객체는 데이터와 그 데이터를 처리하는 기능을 함께 묶은 단위라고 이해했다.
- 필드는 객체가 가지는 값이고, 지역 변수는 메서드나 블록 안에서만 사용하는 값이다.
- 문자열 내용 비교에는 `==`가 아니라 `equals()`를 사용해야 한다.
- 생성자는 객체를 만들 때 호출되며, 객체 생성과 동시에 필드 초기화 같은 작업을 수행할 수 있다.
- 접근 제한자는 클래스나 멤버의 접근 범위를 조절해 코드 안정성을 높이는 데 사용한다.
- 캡슐화는 객체의 상태와 동작을 하나로 묶고, 필요한 부분만 외부에 공개하는 방식이라고 이해했다.
