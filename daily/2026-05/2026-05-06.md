## 오늘 배운 것

### Git 원격 저장소와 협업

- GitHub는 Git 저장소를 원격에서 관리하고 협업할 수 있게 해주는 코드 호스팅 서비스라고 이해했다.
- 로컬 저장소는 개인 작업 공간이고, 원격 저장소는 공유와 협업을 위한 공간이다.
- `git push`는 로컬 커밋을 원격 저장소에 올리는 명령어이다.
- `git fetch`는 원격 저장소의 최신 정보를 가져오지만 현재 브랜치에 바로 병합하지 않는다.
- `git pull`은 보통 `fetch`와 `merge`를 함께 수행하므로 충돌이 발생할 수 있다.
- Feature Branch Workflow는 작업 브랜치를 분리해 `main` 브랜치의 안정성을 유지하는 방식이다.
- Pull Request는 변경사항을 병합하기 전에 리뷰하고 논의할 수 있는 협업 흐름이다.

### Java 기초 문법

- Java는 소스 코드를 바이트코드로 컴파일한 뒤 JVM에서 실행하는 언어이다.
- 변수는 값을 저장하는 공간에 이름을 붙인 것이고, Java에서는 변수 선언 시 자료형이 필요하다.
- 형변환은 자동 형변환과 명시적 형변환으로 나뉘며, 명시적 형변환에서는 데이터 손실을 주의해야 한다.
- 조건문과 반복문은 프로그램의 실행 흐름을 제어하는 기본 문법이다.
- `break`, `continue`, `return`은 반복문이나 메서드의 흐름을 중간에 제어할 때 사용한다.

### Java 파일 I/O와 Stream

- `File` 클래스는 파일 내용 입출력 자체가 아니라 경로, 메타데이터, 생성, 삭제, 목록 조회 등을 다루는 클래스이다.
- 실제 파일 내용 입출력은 Stream, Reader, Writer, Channel이 담당한다.
- `InputStream`과 `OutputStream`은 바이트 기반 입출력에 사용한다.
- `Reader`와 `Writer`는 문자 기반 입출력에 사용하며, 텍스트 데이터에서는 인코딩을 함께 고려해야 한다.
- 보조 스트림은 기본 스트림을 감싸서 버퍼링, 기본형 데이터 처리, 객체 직렬화 같은 기능을 추가한다.
- 직렬화는 객체를 바이트 스트림으로 바꾸는 과정이고, `transient`는 직렬화 대상에서 특정 필드를 제외할 때 사용한다.

### NIO와 네트워크

- NIO는 Buffer와 Channel을 중심으로 동작하는 입출력 API이다.
- Buffer는 데이터를 임시로 저장하는 메모리 영역이고, `position`, `limit`, `capacity` 같은 상태 값을 가진다.
- Channel은 입출력을 수행하는 통로이지만, 모든 Channel이 양방향은 아니며 종류와 open 옵션에 따라 읽기와 쓰기 가능 여부가 달라질 수 있다.
- Selector는 여러 채널의 입출력 준비 상태를 하나의 스레드에서 감지할 수 있게 해준다.
- Java의 `Socket`은 클래스이며, 개념적으로는 OS 소켓 API를 사용할 수 있게 해주는 추상화로 이해할 수 있다.
- Port 번호는 하나의 컴퓨터 안에서 어떤 네트워크 서비스와 통신할지 구분하기 위한 번호로 이해했다.
- TCP는 연결 기반 프로토콜이며, 데이터가 연속된 byte stream으로 전달되기 때문에 애플리케이션에서 메시지 단위를 구분해야 한다.
- 서버 구조는 단일 스레드, 클라이언트별 멀티스레드, NIO 기반 I/O 멀티플렉싱 방식으로 나누어 이해할 수 있다.

## 어려웠던 점

- Buffered I/O와 Blocking I/O가 서로 다른 기준의 개념이라는 점이 어려웠다.
- NIO의 Buffer, Channel, Selector 구조가 기존 Stream 방식과 달라 낯설었다.
- 단일 스레드 서버, 멀티스레드 서버, I/O 멀티플렉싱 서버의 차이를 구분하는 데 시간이 필요했다.

## 해결 방법

- Buffer는 성능을 위해 데이터를 모아 처리하는 메모리 공간이고, Blocking은 작업 완료 전까지 스레드가 기다리는 실행 방식으로 구분했다.
- NIO는 Stream이 아니라 Buffer와 Channel 중심으로 동작한다고 정리했다.
- 서버 구조는 처리 방식에 따라 단일 스레드, 클라이언트별 멀티스레드, Selector 기반 I/O 멀티플렉싱으로 나누어 이해했다.
