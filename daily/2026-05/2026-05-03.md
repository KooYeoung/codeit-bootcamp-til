## 오늘 배운 것

### Java 학습 전 기본기

- Java를 배우기 전에 컴퓨터 구조, 운영체제, 파일 시스템, 인코딩 같은 기본 개념을 이해하는 것이 중요하다.
- 프로그램 실행은 결국 CPU가 메모리의 값을 가져와 연산하고, 그 결과를 다시 메모리에 저장하는 과정이다.
- 개발환경을 다루기 위해 파일, 폴더, 경로, 콘솔 명령어 개념도 필요하다.

### 프로그래밍 기본 개념

- 프로그래밍은 절차적 순서를 논리적으로 작성하는 글쓰기라고 볼 수 있다.
- 코드를 바로 작성하기보다 문제를 정의하고 해결 순서를 먼저 정리하는 습관이 중요하다.
- 변수는 값이 변할 수 있는 데이터이고, 상수는 값이 변하지 않는 데이터이다.

### 컴파일러와 인터프리터

- 컴파일러는 소스코드를 실행 가능한 형태로 번역하는 프로그램이다.
- 인터프리터는 소스코드를 한 줄씩 해석하면서 실행하는 방식이다.
- Java는 `.java` 파일을 `.class` 바이트코드로 컴파일하고, JVM에서 실행한다.

### JDK, JRE, JVM

- JDK는 Java 프로그램 개발에 필요한 도구 모음이다.
- JRE는 Java 프로그램 실행에 필요한 환경이다.
- JVM은 Java 바이트코드를 실행하는 가상 머신이다.
- 정리하면 JDK는 개발환경, JRE는 실행환경, JVM은 실행 주체로 볼 수 있다.

### 빌드타임과 런타임

- 빌드타임에는 `.java` 파일이 `.class` 바이트코드로 변환된다.
- 런타임에는 JVM이 `.class` 파일을 로딩하고 실행한다.
- Java는 컴파일 과정과 JVM 실행 과정을 함께 이해해야 한다.

### Java 자료형

- Java의 기본 자료형에는 정수형, 실수형, 문자형, 논리형이 있다.
- 정수형에는 `byte`, `short`, `char`, `int`, `long`이 있다.
- 실수형에는 `float`, `double`이 있다.
- 논리형은 `boolean`이며 `true`, `false` 값을 가진다.
- 참조 자료형에는 `String`, 배열, 클래스 등이 있다.

### 부동소수점 오차

- `float`, `double`은 실수를 근사값으로 표현한다.
- 모든 소수를 정확히 표현할 수 없기 때문에 실수 연산에서는 오차가 발생할 수 있다.
- 정확한 비교가 필요한 경우 단순히 `==`로 비교하는 방식은 주의해야 한다.

### 문자와 문자열

- `char`는 한 문자를 저장하는 자료형이다.
- `char`는 정수형에 속하므로 문자와 숫자 간 변환이 가능하다.
- `String`은 문자열을 다루는 클래스이다.
- Java의 문자는 Unicode 기반으로 처리된다.
- UTF-8, UTF-16 같은 인코딩 방식에 따라 문자가 바이트로 표현되는 방식이 달라질 수 있다.

### 변수와 상수

- 변수는 값이 정해지지 않았거나 변경될 가능성이 있는 값을 저장하는 공간이다.
- 상수는 값이 변하지 않는 데이터이다.
- Java에서 심볼릭 상수는 `final`을 사용해 표현할 수 있다.
- 변수 이름은 의미 있게 작성해야 하며, 예약어를 사용할 수 없다.
- 의미 없는 변수명은 나중에 코드를 이해하기 어렵게 만든다.

### 주석

- 주석은 코드에 설명을 남기기 위한 문법이다.
- 한 줄 주석은 `//`를 사용한다.
- 여러 줄 주석은 `/* */`를 사용한다.
- 주석은 코드와 동기화되어야 하며, 실제 코드와 다른 주석은 오히려 혼란을 줄 수 있다.

### 콘솔 입출력

- 콘솔 입력은 키보드 입력을 기반으로 동작한다.
- `System.in.read()`는 입력 값을 바이트 단위로 읽는다.
- `Scanner`를 사용하면 정수, 실수, 문자열 입력을 쉽게 처리할 수 있다.
- `nextInt()`, `nextDouble()`, `nextLine()`은 입력 처리 방식이 다르므로 입력 버퍼에 주의해야 한다.

### 조건문, 반복문, 배열

- 조건문은 경우의 수에 따라 프로그램 흐름을 제어할 때 사용한다.
- 반복문은 같은 작업을 여러 번 수행할 때 사용한다.
- 배열은 같은 자료형의 값을 여러 개 묶어서 관리하는 자료구조이다.
- Java에서 배열은 객체이며, 배열의 인덱스는 0부터 시작한다.
- 배열의 마지막 인덱스는 `배열 길이 - 1`이다.

## 어려웠던 점

- Java 문법뿐만 아니라 컴퓨터 구조, 메모리, 인코딩 개념까지 함께 나와서 학습 범위가 넓게 느껴졌다.
- JDK, JRE, JVM의 역할이 비슷해 보여 처음에는 구분이 어려웠다.
- `float`과 `double`은 표현할 수 있는 정밀도에 차이가 있고, 저장 가능한 유효 자릿수를 넘어가거나 2진수로 정확히 표현되지 않는 소수에서는 오차가 발생할 수 있다는 점이 헷갈렸다.

## 해결 방법

- Java 문법만 외우기보다 컴퓨터가 프로그램을 실행하는 흐름과 연결해서 이해하려고 했다.
- JDK는 개발 도구, JRE는 실행 환경, JVM은 바이트코드를 실행하는 가상 머신으로 구분했다.
- 부동소수점 오차는 단순히 자리수가 부족해서만 발생하는 것이 아니라, 10진수 소수를 2진수로 정확히 표현하지 못하는 경우가 있기 때문에 발생한다고 정리했다.
- `float`은 대략 7자리, `double`은 대략 15자리 정도의 유효 자릿수를 가지므로 정밀한 계산이 필요할 때는 자료형 선택과 비교 방식에 주의해야 한다.
