## 1. 객체지향 프로그래밍

객체지향 프로그래밍은 프로그램을 객체 단위로 구성하는 방식이다.

객체는 관련 있는 데이터와 기능을 하나로 묶은 단위이다.

Java에서는 클래스를 사용해 객체를 정의한다.

클래스는 객체를 만들기 위한 설계도이자 사용자 정의 자료형으로 볼 수 있다.

---

## 2. 클래스, 객체, 인스턴스

클래스는 객체를 표현하기 위한 문법이다.

객체는 프로그램에서 다루려는 대상이다.

인스턴스는 클래스를 바탕으로 실제 메모리에 생성된 객체이다.

정리하면 다음과 같다.

- 클래스: 객체를 만들기 위한 설계도
- 객체: 프로그램에서 다루는 대상
- 인스턴스: 클래스로부터 생성된 실제 객체

Java에서 클래스 형식의 변수는 객체 자체를 담는 것이 아니라 객체를 가리키는 참조값을 가진다.

참조형 변수는 `null`을 가질 수 있다.

`null` 상태에서 멤버에 접근하면 `NullPointerException`이 발생할 수 있다.

---

## 3. 클래스의 구성 요소

클래스는 주로 필드, 메서드, 생성자로 구성된다.

- 필드: 객체가 가지는 데이터
- 메서드: 객체가 수행할 수 있는 기능
- 생성자: 객체 생성 시 초기화를 담당하는 특수한 메서드

필드는 보통 `private`으로 감추고, 필요한 기능만 메서드로 제공한다.

이렇게 하면 객체 내부 구현을 외부에서 직접 건드리지 못하게 할 수 있다.

---

## 4. 생성자

생성자는 객체가 생성될 때 자동으로 호출된다.

생성자의 이름은 클래스 이름과 동일하다.

생성자는 반환 자료형이 없다.

생성자를 하나도 작성하지 않은 경우에만 컴파일러가 기본 생성자를 추가한다.

생성자는 다중 정의할 수 있다.

즉, 매개변수 구성이 다르면 여러 생성자를 만들 수 있다.

생성자 안에서 다른 생성자를 호출할 때는 `this()`를 사용할 수 있다.

단, `this()`는 생성자 내부에서 가장 먼저 호출되어야 한다.

---

## 5. 접근 제어 지시자

접근 제어 지시자는 클래스나 멤버에 접근할 수 있는 범위를 제한한다.

- `public`: 외부 접근 허용
- `protected`: 같은 패키지 또는 자식 클래스 접근 허용
- `default`: 같은 패키지 안에서만 접근 허용
- `private`: 클래스 내부에서만 접근 허용

접근 제어를 사용하는 이유는 외부에서 알 필요 없는 구현을 감추고, 사용해야 하는 기능만 제공하기 위해서이다.

---

## 6. getter와 setter

getter와 setter는 `private` 필드에 접근하기 위해 제공하는 메서드이다.

getter는 값을 읽을 때 사용한다.

setter는 값을 변경할 때 사용한다.

필드에 직접 접근하지 않고 getter/setter를 사용하면 값 검증이나 접근 제한을 적용할 수 있다.

예를 들어 나이는 음수가 될 수 없으므로 setter에서 음수 입력을 막을 수 있다.

---

## 7. this

`this`는 현재 객체 자신을 가리키는 참조자이다.

필드와 지역변수의 이름이 같을 때 `this`를 사용하면 객체의 필드임을 명확히 할 수 있다.

예시:

```java
this.data = data;
```

같은 클래스로 만든 객체라도 각 객체의 `this`는 서로 다른 인스턴스를 가리킨다.

`static` 메서드에서는 인스턴스가 없기 때문에 `this`를 사용할 수 없다.

---

## 8. static

`static` 멤버는 인스턴스가 없어도 클래스 기준으로 존재한다.

`static` 필드는 모든 인스턴스가 공유한다.

`static` 메서드는 인스턴스 없이 호출할 수 있다.

하지만 `static` 메서드는 특정 객체에 속한 메서드가 아니기 때문에 `this`를 사용할 수 없다.

또한 인스턴스 필드에 직접 접근할 수도 없다.

정리하면 다음과 같다.

- 인스턴스 멤버: 객체마다 따로 존재
- static 멤버: 클래스 기준으로 하나만 존재
- static 메서드: this 사용 불가
- static final: 상수 정의에 자주 사용

---

## 9. 얕은 복사와 깊은 복사

### 얕은 복사

얕은 복사는 참조 대상 객체는 그대로 두고 참조값만 복사하는 방식이다.

이 경우 두 변수가 같은 객체를 바라보게 된다.

따라서 한쪽에서 객체를 수정하면 다른 쪽에도 영향을 줄 수 있다.

### 깊은 복사

깊은 복사는 참조 대상 객체까지 새로 만들어 값을 복사하는 방식이다.

두 객체가 서로 다른 인스턴스를 가지므로 한쪽을 수정해도 다른 쪽에 영향을 주지 않는다.

참조형 필드를 가진 객체를 복사할 때는 얕은 복사로 인한 사이드 이펙트를 주의해야 한다.

필요한 경우 복사 생성자나 별도의 deep copy 메서드를 만들 수 있다.

---

## 10. 복사 생성자

복사 생성자는 같은 클래스의 다른 인스턴스를 받아 새 객체를 만드는 생성자이다.

참조형 필드가 있을 경우 단순히 참조만 복사하면 얕은 복사가 되므로 주의해야 한다.

깊은 복사가 필요하다면 복사 생성자 안에서 참조 대상 객체까지 새로 만들어야 한다.

---

## 11. 상속

상속은 기존 클래스를 확장해 새로운 클래스를 만드는 문법이다.

Java에서는 `extends`를 사용한다.

부모 클래스와 자식 클래스, 또는 기본 클래스와 파생 클래스라고 부른다.

상속은 `is-a` 관계로 설명할 수 있다.

예를 들어 Triangle은 Shape이다.

Java의 모든 클래스는 최상위에서 Object 클래스를 상속한다.

Java는 클래스의 다중 상속을 허용하지 않는다.

---

## 12. 상속과 생성자 호출

자식 클래스 객체가 생성될 때 부모 클래스 객체도 함께 생성된다.

생성자 호출 흐름은 다음과 같이 이해할 수 있다.

1. 자식 클래스 생성자가 호출된다.
2. 부모 클래스 생성자가 먼저 실행된다.
3. 부모 클래스 초기화가 끝난 뒤 자식 클래스 생성자 내용이 실행된다.

즉, 실제 실행 순서는 부모 생성자 먼저, 자식 생성자 나중이다.

---

## 13. super

`super`는 부모 클래스의 멤버나 생성자를 가리킬 때 사용한다.

부모 클래스 생성자를 명시적으로 호출하려면 `super()`를 사용한다.

예시:

```java
super();
super(param);
```

부모 클래스의 메서드를 호출할 때도 사용할 수 있다.

```java
super.setData(param);
```

오버라이딩한 메서드에서 부모의 기존 기능을 유지하면서 추가 기능을 넣고 싶을 때 유용하다.

---

## 14. 오버로딩과 오버라이딩

### 오버로딩

오버로딩은 같은 이름의 메서드나 생성자를 매개변수 구성을 다르게 해서 여러 개 정의하는 것이다.

### 오버라이딩

오버라이딩은 부모 클래스의 메서드를 자식 클래스에서 재정의하는 것이다.

오버라이딩할 때는 `@Override` 어노테이션을 사용하면 실수를 줄일 수 있다.

정리하면 다음과 같다.

- 오버로딩: 같은 이름, 다른 매개변수
- 오버라이딩: 부모 메서드를 자식 클래스에서 재정의

## 15. 객체 간 관계와 UML

클래스는 다른 클래스를 필드로 가질 수 있다.

객체 간 관계에는 여러 종류가 있다.

- 상속
- 의존
- 연관
- 집합
- 합성

클래스 설계는 객체와 객체 사이의 관계를 정의하는 과정이다.

UML은 클래스와 객체 간 관계를 시각적으로 표현하기 위한 모델링 언어이다.

대표적으로 클래스 다이어그램과 시퀀스 다이어그램이 있다.

---

## 16. 다형성

다형성은 하나의 부모 타입으로 여러 자식 타입 객체를 다룰 수 있는 객체지향의 특징이다.

부모 타입 참조 변수는 자식 클래스 인스턴스를 참조할 수 있다.

이를 업캐스팅이라고 한다.

예시:

```java
Shape shape = new Triangle();
```

부모 타입으로 참조하더라도 오버라이딩된 메서드는 실제 인스턴스 타입 기준으로 실행된다.

다형성을 사용하면 코드 구조를 더 단순하고 유연하게 만들 수 있다.

---

## 17. 다운캐스팅과 instanceof

다운캐스팅은 부모 타입 참조를 자식 타입으로 변환하는 것이다.

잘못된 다운캐스팅은 런타임 오류를 발생시킬 수 있다.

`instanceof`를 사용하면 객체가 특정 타입인지 확인할 수 있다.

예시:

```java
if (shape instanceof Rectangle) {
    Rectangle rect = (Rectangle) shape;
}
```

다운캐스팅이 필요한 경우에는 먼저 `instanceof`로 확인하는 것이 안전하다.

다만 부모 클래스가 자식 타입을 지나치게 많이 판단하는 구조는 의존성이 역전될 수 있으므로 주의해야 한다.

## 18. 추상 클래스

추상 클래스는 직접 인스턴스를 만들 수 없는 클래스이다.

추상 클래스는 자식 클래스를 위한 설계 기준 역할을 한다.

추상 메서드를 가질 수 있고, 자식 클래스가 일반 클래스라면 추상 메서드를 반드시 구현해야 한다.

자식 클래스도 `abstract`라면 구현을 미룰 수 있다.

예시:

```java
abstract class Shape {
    abstract void render();
}
```

추상 클래스는 공통 필드나 공통 메서드도 함께 가질 수 있다.

---

## 19. 인터페이스

인터페이스는 구현 없이 메서드의 형식을 정의하는 역할을 한다.

클래스는 인터페이스를 `implements`로 구현한다.

예시:

```java
interface Shape {
    void render();
}
```

인터페이스는 다중 구현이 가능하다.

인터페이스는 여러 클래스가 같은 규칙을 따르도록 만들 때 사용한다.

---

## 20. 추상 클래스와 인터페이스 차이

추상 클래스는 공통 상태와 기능을 함께 제공할 수 있다.

인터페이스는 구현 규칙을 정의하는 데 초점이 있다.

간단히 정리하면 다음과 같다.

- 추상 클래스: 공통 필드와 공통 기능을 포함할 수 있음
- 인터페이스: 구현해야 할 동작의 규칙을 정의함

## 21. enum

`enum`은 관련 있는 상수들을 하나의 타입으로 묶어 표현하는 문법이다.

단순 문자열이나 숫자 상수를 사용하는 것보다 가독성과 타입 안정성을 높일 수 있다.

자주 사용하는 메서드는 다음과 같다.

- `values()`: 모든 enum 값 반환
- `valueOf()`: 문자열로 enum 값 찾기
- `name()`: enum 상수 이름 반환
- `ordinal()`: enum 상수 순서 반환

## 22. JVM 클래스 로딩

JVM은 `.class` 파일을 읽어 실행 가능한 상태로 준비한다.

클래스 로딩 과정은 크게 다음 흐름으로 볼 수 있다.

```text
Loading
→ Linking
→ Initialization
→ Using
→ Unloading
```

클래스 로더는 클래스의 바이트 스트림을 읽어 JVM이 사용할 수 있는 형태로 준비한다.

Java의 클래스 로딩과 링킹은 런타임에 이루어지며, 이 구조 덕분에 Java는 확장성과 유연성을 가질 수 있다.

---

## 23. Runtime Data Area

Runtime Data Area는 JVM이 프로그램 실행 중 사용하는 메모리 구조이다.

대표적인 영역은 다음과 같다.

- Method Area
- Heap
- Stack
- PC Register
- Native Method Stack

### Method Area

클래스 정보, 상수, static 변수 정보 등이 저장된다.

### Heap

`new`로 생성된 객체가 저장된다.

Garbage Collector의 관리 대상이다.

### Stack

메서드 호출 시 지역변수, 매개변수, 호출 정보 등이 저장된다.

메서드 호출이 끝나면 해당 스택 프레임은 제거된다.

정리하면 다음과 같다.

- 클래스 정보와 static 정보: Method Area
- 객체: Heap
- 메서드 호출과 지역변수: Stack

---

## 24. Object 클래스

Java의 모든 클래스는 Object 클래스를 상속한다.

Object 클래스의 주요 메서드는 다음과 같다.

- `equals()`
- `hashCode()`
- `toString()`
- `getClass()`

`toString()`은 기본적으로 클래스명과 해시코드 형태의 문자열을 반환한다.

`equals()`는 객체 비교에 사용되며, 필요한 경우 클래스에서 재정의해서 논리적 동등성 비교를 구현할 수 있다.

---

## 25. 동일성과 동등성

동일성은 두 참조가 같은 객체를 가리키는지 비교하는 것이다.

Java에서는 `==`로 비교한다.

동등성은 두 객체의 내용이 논리적으로 같은지 비교하는 것이다.

보통 `equals()`를 사용한다.

정리하면 다음과 같다.

- `==`: 참조값 비교
- `equals()`: 내용 비교 목적으로 사용

단, `equals()`가 재정의되어 있지 않으면 기본적으로 참조 비교와 비슷하게 동작할 수 있다.

## 26. String

Java에서 문자열은 `String` 클래스로 다룬다.

`String`은 참조형 자료형이며, 문자열 객체를 가리키는 참조값을 가진다.

또한 `String`은 불변 객체이다.  
즉, 한 번 생성된 문자열 객체의 내용은 직접 변경되지 않는다.

문자열을 비교할 때 `==`는 참조 비교이고, `equals()`는 내용 비교이다.

따라서 문자열 내용 비교에는 보통 `equals()`를 사용해야 한다.

### 문자열 리터럴

문자열 리터럴은 JVM의 문자열 상수 풀에서 관리될 수 있다.

```java
String a = "Hello";
String b = "Hello";

System.out.println(a == b);      // true
System.out.println(a.equals(b)); // true
```

위 코드에서 `a == b`가 `true`인 이유는 문자열 내용이 같아서가 아니라,  
두 참조가 문자열 상수 풀에 있는 같은 `"Hello"` 객체를 가리킬 수 있기 때문이다.

### new String()

`new`를 사용하면 새로운 `String` 객체를 생성한다.

```java
String a = "Hello";
String b = new String("Hello");

System.out.println(a == b);      // false
System.out.println(a.equals(b)); // true
```

`a`와 `b`는 문자열 내용은 같지만 서로 다른 객체를 참조한다.  
그래서 `==` 비교는 `false`가 되고, 내용 비교인 `equals()`는 `true`가 된다.

정리하면 다음과 같다.

- `==`: 두 참조가 같은 객체를 가리키는지 비교
- `equals()`: 문자열의 내용이 같은지 비교
- 문자열 내용 비교에는 보통 `equals()`를 사용
- 문자열 리터럴은 문자열 상수 풀에서 재사용될 수 있음
- `new String()`은 내용이 같아도 새로운 객체를 생성

## 28. 예외 처리

예외는 정상적인 실행이 불가능한 예외 상황을 의미한다.

예외 처리를 사용하면 오류 상황을 별도로 처리하고 프로그램 흐름을 이어갈 수 있다.

Java에서는 `try`, `catch`, `finally`를 사용한다.

예시:

```java
try {
    // 예외가 발생할 수 있는 코드
} catch (Exception e) {
    // 예외 처리
} finally {
    // 예외 발생 여부와 관계없이 실행
}
```

### Checked Exception

컴파일 시점에 예외 처리를 강제하는 예외이다.

`catch` 또는 `throws` 처리가 필요하다.

### Unchecked Exception

런타임에 발생할 수 있는 예외이다.

예외 처리를 강제하지 않는다.

대표적인 런타임 예외는 다음과 같다.

- `NullPointerException`
- `ArrayIndexOutOfBoundsException`
- `NumberFormatException`
- `ClassCastException`

## 29. 오늘 정리

오늘은 Java 객체지향의 핵심 개념을 학습했다.

가장 중요한 흐름은 다음과 같다.

```text
클래스 이해
→ 객체와 인스턴스 이해
→ 생성자와 접근 제어 이해
→ this와 static 구분
→ 복사 방식 이해
→ 상속과 오버라이딩 학습
→ 다형성 이해
→ 추상 클래스와 인터페이스 학습
→ JVM 구조 이해
→ String과 예외 처리 학습
```

오늘 학습에서 가장 중요한 점은 Java의 클래스 형식은 객체 자체가 아니라 참조를 다룬다는 것이다.

참조 개념을 이해해야 `null`, 얕은 복사, 깊은 복사, 다형성, String 비교, 예외 처리까지 연결해서 이해할 수 있다.
