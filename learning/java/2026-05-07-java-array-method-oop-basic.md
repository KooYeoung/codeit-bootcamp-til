## 1. 반복문에서 continue 흐름

`continue`는 현재 반복에서 남은 코드를 건너뛰고 다음 반복으로 넘어갈 때 사용한다.

`for`문에서는 `continue`를 만나면 증감식으로 이동한 뒤 조건식을 다시 검사한다.

`while`문에서는 `continue`를 만나면 바로 조건식으로 이동한다.

그래서 `while`문에서 `continue`를 사용할 때는 조건식에 영향을 주는 값이 갱신되는지 특히 주의해야 한다.

갱신 없이 `continue`가 실행되면 무한 반복이 될 수 있다.

## 2. 배열

배열은 같은 타입의 여러 값을 하나의 묶음으로 관리하는 자료구조이다.

Java에서 배열은 한 번 생성하면 길이가 고정된다.

배열의 각 값은 인덱스로 접근하며, 인덱스는 0부터 시작한다.

```java
int[] numbers = new int[5];

numbers[0] = 50;
numbers[1] = 77;
numbers[2] = (int) 3.14;
numbers[3] = numbers[0] + 20;
numbers[4] = 100;
```

`double` 값을 `int` 배열에 넣으려면 명시적 형변환이 필요하다.

이때 소수점 아래 값은 사라지므로 데이터 손실을 주의해야 한다.

배열의 길이는 `length`로 확인한다.

```java
System.out.println(numbers.length);
```

배열을 순회할 때는 일반 `for`문과 향상된 `for`문을 사용할 수 있다.

```java
for (int i = 0; i < numbers.length; i++) {
    System.out.println(numbers[i]);
}

for (int number : numbers) {
    System.out.println(number);
}
```

향상된 `for`문은 배열의 처음부터 끝까지 순회할 때 편리하다.

다만 인덱스가 필요하거나 특정 위치의 값을 수정해야 한다면 일반 `for`문이 더 적합하다.

배열 변수를 그대로 출력하면 배열 내부 값이 아니라 참조값 형태의 문자열이 출력된다.

배열 내용을 보기 위해서는 반복문이나 `Arrays.toString()` 같은 도구를 사용해야 한다.

## 3. 2차원 배열

2차원 배열은 배열 안에 배열이 들어 있는 구조이다.

표처럼 행과 열을 가진 데이터로 생각하면 이해하기 쉽다.

```java
int[][] arr = new int[3][4];
arr[1][1] = 100;
```

Java 배열은 생성될 때 각 타입의 기본값으로 초기화된다.

- 정수형: `0`
- 실수형: `0.0`
- 논리형: `false`
- 문자형: 공백 문자
- 참조형: `null`

2차원 배열을 출력할 때는 중첩 반복문을 사용할 수 있다.

```java
for (int[] row : arr) {
    for (int value : row) {
        System.out.print(value + " ");
    }
    System.out.println();
}
```

2차원 배열도 결국 배열의 요소가 다시 배열인 구조라고 이해했다.

## 4. 메서드

메서드는 반복되는 코드 블록에 이름을 붙여 재사용할 수 있게 만든 것이다.

메서드를 사용하면 코드 중복을 줄일 수 있고, 기능 단위로 코드를 나눌 수 있어 가독성이 좋아진다.

메서드는 매개변수를 통해 외부 값을 전달받고, 필요하면 실행 결과를 반환한다.

```java
int add(int a, int b) {
    return a + b;
}
```

매개변수는 메서드 실행에 필요한 값을 전달하는 통로이다.

반환값은 메서드 실행 결과로 돌려주는 값이며, 반환값은 하나만 가질 수 있다.

반환할 값이 없는 메서드는 `void`를 사용한다.

```java
void printName(String name) {
    System.out.println(name);
}
```

`return`은 값을 반환할 때 사용하지만, 메서드 실행을 중간에 종료하는 역할도 한다.

`void` 메서드에서는 `return;`처럼 값을 반환하지 않고 종료할 수 있다.

## 5. 객체지향 프로그래밍 입문

객체는 데이터와 그 데이터를 처리하는 기능을 함께 묶은 하나의 단위이다.

예를 들어 펜을 프로그램에서 표현한다면 색상, 굵기 같은 속성과 쓰기 같은 기능을 함께 생각할 수 있다.

객체를 사용하지 않으면 비슷한 속성과 기능을 가진 대상이 늘어날 때 변수와 메서드가 계속 늘어난다.

객체지향 방식은 관련 있는 데이터와 기능을 클래스라는 설계도로 묶고, 필요한 만큼 객체를 만들어 사용하는 방식이라고 이해했다.

## 6. 필드와 지역 변수

필드와 지역 변수는 모두 값을 저장하는 변수이지만, 선언 위치와 동작 방식이 다르다.

필드는 클래스 내부에 선언되며 객체의 상태를 나타낸다.

지역 변수는 메서드나 블록 내부에 선언되며 해당 영역 안에서만 사용할 수 있다.

| 구분 | 필드 | 지역 변수 |
| --- | --- | --- |
| 선언 위치 | 클래스 내부 | 메서드 또는 블록 내부 |
| 사용 범위 | 객체 내부의 여러 메서드 | 선언된 블록 내부 |
| 초기화 | 기본값으로 자동 초기화 | 직접 초기화해야 사용 가능 |
| 메모리 | 객체 생성 시 객체마다 생성 | 블록 실행 중 스택에 생성 |

필드는 객체가 생성될 때 객체마다 별도의 값으로 존재한다.

지역 변수는 블록이 실행될 때 생기고, 블록이 끝나면 사라진다고 이해했다.

## 7. 문자열 비교와 equals()

Java에서 문자열을 비교할 때는 `==`와 `equals()`를 구분해야 한다.

`==`는 두 변수가 같은 객체를 참조하는지 비교한다.

`equals()`는 문자열의 내용이 같은지 비교한다.

```java
String a = "java";
String b = new String("java");

System.out.println(a == b);
System.out.println(a.equals(b));
```

문자열 리터럴은 문자열 상수 풀에 저장되기 때문에, 같은 문자열을 재사용하면 `==` 결과가 `true`처럼 보일 수 있다.

하지만 `new String()`으로 새 객체를 만들면 내용이 같아도 참조값은 달라질 수 있다.

그래서 문자열의 내용 비교가 목적이라면 `equals()`를 사용해야 한다고 정리했다.

## 8. 생성자

생성자는 객체를 생성할 때 호출되는 특별한 메서드와 비슷한 구조이다.

생성자의 이름은 클래스 이름과 완전히 같아야 하며, 리턴 타입을 작성하지 않는다.

```java
class Pen {
    String color;

    Pen(String color) {
        this.color = color;
    }
}
```

생성자는 객체를 만드는 역할뿐 아니라, 객체가 처음 만들어질 때 필요한 초기화 작업을 함께 수행할 수 있다.

생성자를 하나도 작성하지 않으면 JVM이 기본 생성자를 자동으로 제공한다.

하지만 매개변수가 있는 생성자를 직접 작성하면 기본 생성자는 자동으로 만들어지지 않는다.

이 경우 기본 생성자가 필요하다면 직접 작성해야 한다.

## 9. 패키지

패키지는 비슷한 성격의 Java 클래스나 인터페이스를 모아 놓는 디렉토리 구조이다.

패키지를 사용하면 클래스들을 목적에 따라 정리할 수 있고, 같은 이름의 클래스가 충돌하는 문제도 줄일 수 있다.

패키지 선언은 Java 파일의 가장 위에 작성한다.

```java
package com.example.study;
```

패키지 이름은 보통 소문자로 작성하고, 상위 패키지와 하위 패키지는 점으로 구분한다.

패키지 이름에도 식별자 규칙이 적용되므로 Java 키워드나 숫자로 시작하는 이름은 사용할 수 없다.

## 10. this와 this()

`this`는 현재 객체 자신을 가리키는 키워드이다.

필드 이름과 매개변수 이름이 같을 때 `this`를 사용하면 둘을 구분할 수 있다.

```java
class Member {
    String name;

    Member(String name) {
        this.name = name;
    }
}
```

위 코드에서 `this.name`은 객체의 필드이고, 오른쪽 `name`은 생성자의 매개변수이다.

`this()`는 같은 클래스 안의 다른 생성자를 호출할 때 사용한다.

```java
class Member {
    String name;
    int age;

    Member() {
        this("unknown", 0);
    }

    Member(String name, int age) {
        this.name = name;
        this.age = age;
    }
}
```

`this()`는 반드시 생성자의 첫 줄에 작성해야 한다.

다른 생성자에게 공통 초기화 로직을 맡기고, 현재 생성자에서 필요한 나머지 작업을 이어서 작성하는 방식으로 이해했다.

## 11. 접근 제한자

접근 제한자는 클래스, 필드, 메서드, 생성자에 접근할 수 있는 범위를 정하는 키워드이다.

대표적인 접근 제한자는 다음과 같다.

| 접근 제한자 | 접근 범위 |
| --- | --- |
| `public` | 모든 패키지에서 접근 가능 |
| `protected` | 같은 패키지 또는 다른 패키지의 자식 클래스에서 접근 가능 |
| `default` | 같은 패키지 안에서만 접근 가능 |
| `private` | 같은 클래스 안에서만 접근 가능 |

아무 접근 제한자도 작성하지 않으면 `default` 접근 제한이 적용된다.

클래스 자체에는 `public`과 `default`만 사용할 수 있고, 일반 클래스에는 `private`이나 `protected`를 붙일 수 없다.

접근 제한자를 사용하는 이유는 외부에서 접근할 필요가 없거나 접근하면 안 되는 부분을 숨겨 코드의 안정성을 높이기 위해서라고 이해했다.

## 12. 정보 은닉과 캡슐화

정보 은닉은 객체의 상태를 외부에서 직접 접근하지 못하도록 숨기는 것이다.

캡슐화는 관련 있는 상태와 동작을 하나의 단위로 묶고, 외부에는 필요한 기능만 공개하는 방식이다.

예를 들어 필드는 `private`으로 숨기고, 필요한 경우 메서드를 통해 값을 읽거나 변경하게 만들 수 있다.

```java
class Account {
    private int balance;

    void deposit(int amount) {
        if (amount > 0) {
            balance += amount;
        }
    }

    int getBalance() {
        return balance;
    }
}
```

이렇게 하면 외부에서 `balance`를 마음대로 바꾸지 못하고, `deposit()` 메서드의 규칙을 거쳐서만 상태가 변경된다.

캡슐화는 단순히 필드를 숨기는 것이 아니라, 객체가 올바른 상태를 유지하도록 책임을 객체 내부에 두는 방식이라고 이해했다.

## 13. 오늘 정리

오늘은 Java 기초 문법에서 객체지향으로 넘어가기 위한 핵심 개념을 정리했다.

배열과 메서드는 데이터를 묶고 코드를 기능 단위로 나누는 기본 도구이다.

객체지향에서는 데이터와 기능을 객체 단위로 묶고, 생성자와 `this`를 이용해 객체 초기화를 관리한다.

접근 제한자와 캡슐화는 객체 내부 상태를 보호하고, 외부에는 필요한 기능만 제공하기 위한 개념이다.
