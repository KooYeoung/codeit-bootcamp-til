## 1. 탐색 알고리즘

탐색은 여러 데이터 중에서 원하는 값을 찾는 과정이다.

오늘은 대표적인 탐색 방식으로 선형 탐색과 이진 탐색을 학습했다.

### 선형 탐색

선형 탐색은 배열이나 목록을 앞에서부터 하나씩 확인하면서 원하는 값을 찾는 방식이다.

데이터가 정렬되어 있지 않아도 사용할 수 있다는 장점이 있다.

다만 찾는 값이 뒤쪽에 있거나 아예 없다면 모든 값을 확인해야 하므로, 데이터가 많아질수록 시간이 오래 걸릴 수 있다.

```java
int[] numbers = {1, 2, 3, 4, 5};
int target = 4;
int index = -1;

for (int i = 0; i < numbers.length; i++) {
    if (numbers[i] == target) {
        index = i;
        break;
    }
}
```

선형 탐색의 시간 복잡도는 `O(n)`이라고 이해했다.

입력 데이터가 2배가 되면 최악의 경우 확인해야 하는 횟수도 거의 2배로 늘어난다.

### 이진 탐색

이진 탐색은 정렬된 데이터에서 가운데 값을 기준으로 탐색 범위를 절반씩 줄여 나가는 방식이다.

가운데 값이 찾는 값보다 크면 왼쪽 범위를 보고, 작으면 오른쪽 범위를 본다.

이 과정을 반복하면 전체 데이터를 하나씩 확인하지 않아도 값을 찾을 수 있다.

```java
int[] numbers = {1, 2, 3, 4, 5, 6, 7};
int target = 4;

int left = 0;
int right = numbers.length - 1;

while (left <= right) {
    int mid = (left + right) / 2;

    if (numbers[mid] == target) {
        break;
    } else if (numbers[mid] > target) {
        right = mid - 1;
    } else {
        left = mid + 1;
    }
}
```

이진 탐색의 시간 복잡도는 `O(log n)`이다.

중요한 조건은 데이터가 반드시 정렬되어 있어야 한다는 점이다.

정렬되어 있지 않은 배열에 이진 탐색을 적용하면 가운데 값을 기준으로 범위를 줄이는 판단 자체가 틀릴 수 있다.

## 2. 정렬 알고리즘

정렬은 데이터를 일정한 기준에 따라 순서대로 배치하는 과정이다.

정렬 방식에는 버블 정렬, 선택 정렬, 삽입 정렬, 병합 정렬, 퀵 정렬, 힙 정렬, 기수 정렬, 버킷 정렬 등이 있다.

오늘은 그중 선택 정렬과 삽입 정렬의 흐름을 중심으로 이해했다.

### 선택 정렬

선택 정렬은 아직 정렬되지 않은 구간에서 가장 작은 값의 위치를 찾고, 현재 위치의 값과 교환하는 방식이다.

예를 들어 첫 번째 자리에는 전체 중 가장 작은 값을 놓고, 두 번째 자리에는 남은 값 중 가장 작은 값을 놓는 식으로 진행된다.

```java
int[] numbers = {5, 3, 1, 7, 2};

for (int i = 0; i < numbers.length; i++) {
    int minIndex = i;

    for (int j = i + 1; j < numbers.length; j++) {
        if (numbers[minIndex] > numbers[j]) {
            minIndex = j;
        }
    }

    if (minIndex != i) {
        int temp = numbers[i];
        numbers[i] = numbers[minIndex];
        numbers[minIndex] = temp;
    }
}
```

선택 정렬은 반복문 안에 반복문이 있기 때문에 보통 `O(n^2)` 시간 복잡도를 가진다.

데이터가 많아질수록 비교 횟수가 빠르게 늘어난다.

### 삽입 정렬

삽입 정렬은 값을 하나씩 적절한 위치에 끼워 넣는 방식이다.

이미 정렬된 구간을 유지하면서, 새 값을 그 구간 안의 올바른 위치에 넣는다고 이해했다.

배열은 크기가 고정되어 있고 중간에 값을 넣으려면 기존 값들을 뒤로 밀어야 하므로, 삽입 과정에서 이동 비용이 발생할 수 있다.

삽입 정렬도 보통 `O(n^2)` 시간 복잡도를 가진다.

## 3. 시간 복잡도

시간 복잡도는 입력 데이터의 크기와 실행 시간의 관계를 표현하는 기준이다.

정확한 실행 시간을 초 단위로 재는 것이 아니라, 입력이 커질 때 실행 시간이 어떤 속도로 증가하는지를 보는 개념이다.

예를 들어 같은 컴퓨터에서 실행하더라도 데이터가 10개일 때와 10만 개일 때의 차이를 예측하기 위해 사용한다.

### Big-O 표기법

Big-O 표기법은 알고리즘의 성능을 표현할 때 자주 사용하는 표기법이다.

가장 중요한 규칙은 입력 크기가 커질 때 가장 영향이 큰 항을 중심으로 본다는 점이다.

대표적인 시간 복잡도는 다음과 같다.

- `O(1)`: 입력 크기와 상관없이 거의 일정한 시간
- `O(log n)`: 입력 크기가 커져도 탐색 범위를 빠르게 줄이는 시간
- `O(n)`: 입력 크기에 비례해 증가하는 시간
- `O(n log n)`: 선형 처리와 로그 처리가 함께 나타나는 시간
- `O(n^2)`: 중첩 반복문처럼 입력 크기의 제곱에 가깝게 증가하는 시간
- `O(n^3)`: 3중 반복문처럼 더 가파르게 증가하는 시간

일반적인 증가 속도는 다음 순서로 이해했다.

```text
O(1) < O(log n) < O(n) < O(n log n) < O(n^2) < O(n^3)
```

### 로그 시간 이해

컴퓨터 과학에서 로그를 말할 때는 보통 밑이 2인 로그를 의미하는 경우가 많다.

이진 탐색처럼 범위를 매번 절반으로 줄이면, 몇 번 나누어야 1에 가까워지는지를 생각하게 된다.

그래서 이진 탐색의 반복 횟수는 `log n`에 가깝다고 이해했다.

## 4. 공간 복잡도

공간 복잡도는 알고리즘이 실행될 때 추가로 사용하는 메모리 양을 표현하는 기준이다.

시간이 빠른 알고리즘이라도 너무 많은 메모리를 사용하면 실제 프로그램에서 문제가 될 수 있다.

오늘은 주로 추가로 생성하는 변수나 자료구조가 메모리 사용량에 영향을 준다고 이해했다.

예를 들어 배열 하나를 그대로 순회하면서 값만 비교하는 경우와, 같은 크기의 배열을 하나 더 만들어 사용하는 경우는 공간 사용량이 다르다.

---

## 5. 헷갈릴 수 있는 부분

선형 탐색은 느릴 수 있지만 정렬이 필요 없다는 장점이 있다.

이진 탐색은 빠르지만 정렬된 데이터에서만 사용할 수 있다.

선택 정렬과 삽입 정렬은 구현 흐름은 단순하지만, 데이터가 많아지면 `O(n^2)` 때문에 비효율적일 수 있다.

시간 복잡도는 실제 실행 시간이 아니라 증가 경향을 보는 기준이다.

공간 복잡도는 프로그램 전체 메모리가 아니라 알고리즘 수행 과정에서 추가로 필요한 메모리에 집중해서 보면 된다.

## 6. 오늘 정리

오늘은 알고리즘을 볼 때 단순히 코드가 동작하는지만 보는 것이 아니라, 입력이 커졌을 때 얼마나 효율적인지도 함께 봐야 한다는 점을 학습했다.

탐색에서는 정렬 여부가 사용할 수 있는 알고리즘을 결정한다.

정렬에서는 반복문의 중첩 구조가 시간 복잡도에 큰 영향을 준다.

앞으로 알고리즘 문제를 풀 때는 먼저 데이터 크기와 정렬 여부를 확인하고, 시간 복잡도를 예상하는 습관이 필요하다고 느꼈다.
